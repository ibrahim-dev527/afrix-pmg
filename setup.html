1
Create Your Supabase Account
Free — no credit card required
  1. Go to supabase.com and click "Start your project"
  2. Sign up using your GitHub account or email
  3. Click "New project"
  4. Fill in:
    • Name: afrix-properties
    • Database Password: choose a strong password and save it
    • Region: choose the closest to Ghana (e.g. West EU or East US)
  5. Click "Create new project" — takes about 60 seconds to set up

The Free tier gives you 500MB database, 1GB storage, and 50,000 monthly active users — more than enough to run a real estate website.

-- 1. PROPERTIES TABLE
CREATE TABLE IF NOT EXISTS properties (
  id           uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  title        text NOT NULL,
  description  text,
  price        numeric NOT NULL DEFAULT 0,
  price_period text DEFAULT 'total',
  location     text NOT NULL,
  city         text DEFAULT 'accra',
  type         text NOT NULL,
  listing_type text NOT NULL,
  bedrooms     integer DEFAULT 0,
  bathrooms    integer DEFAULT 0,
  size         numeric,
  features     jsonb DEFAULT '[]',
  images       jsonb DEFAULT '[]',
  featured     boolean DEFAULT false,
  status       text DEFAULT 'active',
  created_at   timestamptz DEFAULT now(),
  updated_at   timestamptz DEFAULT now()
);

-- 2. LEADS TABLE (contact forms)
CREATE TABLE IF NOT EXISTS leads (
  id          uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  name        text,
  phone       text,
  email       text,
  subject     text,
  message     text,
  lead_type   text DEFAULT 'general',
  property_id uuid,
  status      text DEFAULT 'new',

-- 3. PROPERTY SUBMISSIONS (sell requests)
CREATE TABLE IF NOT EXISTS property_submissions (
  id            uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  owner_name    text,
  phone         text,
  email         text,
  listing_type  text,
  property_type text,
  address       text,
  price         text,
  bedrooms      text,
  description   text,
  status        text DEFAULT 'pending',
  created_at    timestamptz DEFAULT now()
);

-- 4. MANAGEMENT REQUESTS
CREATE TABLE IF NOT EXISTS management_requests (
  id               uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  name             text,
  phone            text,
  email            text,
  property_address text,
  property_type    text,
  message          text,
  status           text DEFAULT 'new',
  created_at       timestamptz DEFAULT now()

-- 5. SITE CONTENT (CMS)
CREATE TABLE IF NOT EXISTS site_content (
  id         uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  page       text NOT NULL,
  section    text NOT NULL,
  title      text,
  content    text,
  updated_at timestamptz DEFAULT now(),
  UNIQUE(page, section)
);

-- 6. MEDIA LIBRARY
CREATE TABLE IF NOT EXISTS media (
  id         uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  filename   text,
  url        text NOT NULL,
  bucket     text DEFAULT 'property-images',
  size       integer,
  created_at timestamptz DEFAULT now()
);

CREATE POLICY "Public read active properties"
  ON properties FOR SELECT USING (status = 'active');

-- PUBLIC: read site content
CREATE POLICY "Public read site_content"
  ON site_content FOR SELECT USING (true);

-- PUBLIC: submit leads/forms
CREATE POLICY "Public insert leads"

CREATE POLICY "Public insert submissions"
  ON property_submissions FOR INSERT WITH CHECK (true);
CREATE POLICY "Public insert management"
  ON management_requests FOR INSERT WITH CHECK (true);

  ON properties FOR ALL USING (auth.role() = 'authenticated');
CREATE POLICY "Admin all leads"

CREATE POLICY "Admin all submissions"

3
Create Image Storage Bucket
Where property photos are stored and served
  1. In your Supabase dashboard, click "Storage" in the left sidebar
  2. Click "New bucket"
  3. Name it exactly: property-images
  4. Toggle "Public bucket" to ON (so images can be viewed by website visitors)
  5. Click "Create bucket"

The bucket name must be exactly property-images (lowercase, with hyphen). This is hardcoded in the admin dashboard and supabase-config.js.

Once created, the admin dashboard can upload images directly. Public URLs are automatically generated and stored in your properties table.

How Images Work

  1. Admin uploads image in admin.html → Media Library or property form
  2. File is uploaded to Supabase Storage: property-images/properties/filename.jpg
  3. Public URL is retrieved: https://xxx.supabase.co/storage/v1/object/public/property-images/...
  4. URL is saved in the properties.images column as a JSON array
  5. Your website reads the URL and displays the image using <img src="...">
4
Paste Your API Keys
The most important step — connects your website to Supabase

Step 4a — Find Your API Keys

  1. In Supabase, click the ⚙️ "Project Settings" icon (bottom of left sidebar)
  2. Click "API"
  3. You'll see two values — copy both:
ValueWhere to find it
Project URLLooks like: https://abcxyz.supabase.co
anon public keyLong string starting with eyJ...

Step 4b — Edit supabase-config.js

  1. Open supabase-config.js in any text editor (Notepad, VS Code, etc.)
  2. Find lines 18–19 and replace the placeholder values:
// BEFORE (placeholder):
const SUPABASE_URL      = 'https://YOUR_PROJECT_ID.supabase.co';
const SUPABASE_ANON_KEY = 'YOUR_ANON_PUBLIC_KEY_HERE';

// AFTER (paste your real values):
const SUPABASE_URL      = 'https://abcxyzexample.supabase.co';
const SUPABASE_ANON_KEY = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';

Save the file after editing. The anon key is safe to use in browser-side code. Never use the service_role secret key in frontend code.

Once you save supabase-config.js with real keys, the website will immediately start loading properties from your Supabase database instead of the static fallback data.

5
Create Your Admin Account
Login credentials for the admin dashboard
  1. In Supabase, click "Authentication" in the left sidebar
  2. Click "Users"
  3. Click "Invite user" or "Add user"
  4. Enter your email address (e.g. admin@afrixproperties.com)
  5. Set a strong password
  6. Click "Create user"

You can add multiple admin users. Anyone with a confirmed account in Supabase Auth can log in to admin.html.

Test the Admin Login

  1. Open admin.html in your browser
  2. Enter the email and password you just created
  3. Click "Sign In to Dashboard"
  4. You should see the full CMS dashboard
  5. Try adding a property using the "Add Property" button
Admin Dashboard Features
  • 📦 Properties Manager — Add, edit, delete properties. Upload images. Set type, price, location, features.
  • 📬 Sell Submissions — Review properties submitted by the public. Approve or reject.
  • 🔑 Management Requests — See landlords who want property management.
  • 👥 Leads — All contact form submissions with WhatsApp quick-reply buttons.
  • ✏️ Content Manager — Edit homepage text, about page content, etc. without touching code.
  • 🖼 Media Library — Upload images, copy URLs, manage your photo library.
6
Deploy to the Internet
Put your website live with a real domain

Option A — Netlify (Recommended, Free)

  1. Go to app.netlify.com → Sign up free
  2. Click "Add new site" → "Deploy manually"
  3. Drag and drop your entire afrix/ folder onto the page
  4. Your site goes live at a URL like afrix-pmg.netlify.app
  5. To add your own domain: Site Settings → Domain management → Add domain

Netlify gives you free HTTPS/SSL automatically. Your site will be accessible at https://afrixproperties.com once you connect your domain.

Option B — Traditional Web Hosting (cPanel)

  1. Log in to your hosting control panel (cPanel or Plesk)
  2. Open File Manager → Navigate to public_html
  3. Upload all files from the afrix/ folder
  4. Your site is live at your domain

After Deployment — Update og:url

Once you have your live URL, update the og:url meta tag on each page:

<meta property="og:url" content="https://afrixproperties.com/index.html"/>

Also update og:image to the full URL of your logo:

<meta property="og:image" content="https://afrixproperties.com/images/logo.png"/>
